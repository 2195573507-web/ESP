## 2. 不可变架构边界

实施必须保留以下现有 ESPC52 机制：

- `lcd_driver`，以及 `lcd_service` 和 `lcd_ui` 的生命周期与失败回滚；
- 当前 LVGL service/timer、`lvgl_port_lock` 使用边界和单一 LVGL task；
- 4,800 B internal-DMA 单 draw buffer；不得增设全帧或双缓冲，也不得将 DMA draw buffer 移入 PSRAM；
- 当前 PSRAM UI arena；
- copied snapshot/value-copy 数据模型；
- `lcd_wake_event_t` wake event queue；
- 触摸降级处理，以及停止、清理和初始化失败回滚路径。

以下行为禁止：复制或替换 ESPLCD C51 的 `lcd.c`，复制或替换 `lcd_lvgl_service.c` 的服务所有权，创建第二个 LVGL task 或新业务 task，在 LVGL callback 中直接调用 `voice_chain_request_local_wake()`，或让非 LVGL task 直接操作 LVGL 对象。

本任务不修改 LCD 引脚、ST7789 参数、分区表、通信协议、雷达业务、ESPS3 或 ESP-server。参考项目只提供视觉、对象组织和交互表现依据，不是服务架构的复制来源。

## 4. 初始事实与设计判据

此前迁移审计确认，主项目 C5 LCD 的资源预算是单一 240 x 10 x RGB565 draw buffer，即 4,800 B internal-DMA；该约束仍是本任务的硬边界。旧 ESPLCD C51 仪表盘包含触摸、环境/网络/语音状态和猫咪交互，但其直接业务耦合不能照搬。

本次同步的设计判据为：

1. ESPC52 的页面视觉层级、Dashboard 构图、像素猫、启动就绪和语音覆盖层尽量接近 ESPLCD C51。
2. 状态来源仍为 ESPC52 已发布的副本快照；UI timer 不得读取 I2C、网络、BLE、雷达或语音业务对象。
3. 点击回调只能构造/提交 wake event；实际语音唤醒由既有 queue consumer 在非 LVGL 回调路径处理。
4. LCD 初始化、页创建、隐藏/显示、停止、对象删除和失败回滚必须保留既有所有权和锁顺序。

### 5.2 Architecture Agent 结论

已确认 C52 UI 的安全改动面是 `lcd_ui.c` 中的对象树、标签格式化、既有 LVGL timer 和
`lcd_ui_stop()` 的对象删除路径；`lcd_driver` 和 `lcd_service` 的服务所有权不得变更。

环境字段必须沿当前副本链路扩展：

```text
bme_sensor_service_reading_t
  -> lcd_system_snapshot_t
  -> c5_lcd_publish_snapshot()
  -> lcd_ui_apply()
```

Cat 点击的既有合规路径为：

```text
LVGL event
  -> lcd_service_request_wake_event()
  -> static depth-1 lcd_wake_event_t queue
  -> c5_lcd_event_task
  -> voice_chain_request_local_wake()
```

因此 LVGL event callback 只能提交事件，不能直接唤醒语音。审查还确认单一 LVGL task、
4,800 B internal-DMA 单 draw buffer、96 KiB PSRAM UI arena、触摸降级以及清理/失败回滚路径均
已存在，实施必须复用而非替换。

### 5.3 Reference Agent 结论

可采用的 C51 视觉规格如下。颜色：背景 `#101820`、标题 `#FFFFFF`、正常/分区薄荷绿
`#79D2A6`、数值 `#DCE7EF`、初始化黄 `#E5B75B`、错误粉红 `#E56B6F`。参考实现未显式指定字体，
以 LVGL 默认字体、全大写 section、固定留白和颜色形成层级。

Dashboard 参考坐标：标题 `SensAir C5` 在 y=12，`ENVIRONMENT` 在 y=44；TEMP/HUM/PRESS/GAS
分别在 y=72/99/126/153，AIR 在 y=182，左边距 x=12、字段宽 144。AIR 状态点在 x=39,y=185；
网络点在 x=82,y=258，网络文字在 x=98,y=253,w=100。网络状态映射为 Online/Connecting/Offline，
对应绿/黄/红。本任务明确要求环境无效值统一为 `--`，优先于参考实现的 `--.-` 或 `----.-`。

Dashboard Pixel Cat 使用 64 x 48、I4 调色板、Flash/rodata 静态帧，参考位置 x=156,y=180；
透明命中区为 x=143,y=169,w=90,h=70，状态文字为 x=156,y=230。参考状态帧包括 listen/open/rec/
play/error；wake 采用 6 帧上下偏移，record 采用 6 帧左右偏移，播放每 4 个 100 ms tick 变更嘴型。
Voice Overlay 使用同一形象的 128 x 96 版本，居中 x=56,y=94；不得引入运行时解码或内部 RAM
长期帧副本。

Boot readiness overlay 参考为 240 x 284 黑色全屏层：96 x 72 Cat 在 x=72,y=48，Display/Sensor/
Network/Audio 行在 y=125/150/175/200，状态点 x=50，进度条 x=40,y=240,w=160,h=8。最终 C52
Boot Cat 已采用该 96 x 72 尺寸和 x=72,y=48 坐标。READY 使用
`#79D2A6`，INIT 使用 `#E5B84B`；C51 的 100 ms 检查、最少 3 s、最多 8 s 仅为视觉时序参考，
C52 readiness 必须来自已有 copied snapshot/view model，不能直接查询 BME、Wi-Fi 或音频服务。

### 5.4 Review Agent 结论

本次 UI 改动符合以下边界：Cat callback 仅经既有 wake queue 传递事件；没有跨任务直接操作
LVGL 对象、第二个 LVGL task、新业务 task、PSRAM DMA draw buffer 或 LVGL callback 对
`voice_chain_request_local_wake()` 的直接调用。UI 只消费 copied snapshot；Voice Overlay 由 voice
state snapshot 控制。`air_quality_valid` 在该审查后补入，避免 Air 数据无效时误显示为有效值。

残余风险均为既有路径，未由本次 UI 改动消除：`lcd_driver` deinit timeout 可能触发 use-after-free，
且 service 当前忽略 driver stop 的失败结果；另外仍存在 PSRAM task stack 与 cache-off 区间的既有
风险。这些风险不能由本次构建验证消除，需后续定向修复和硬件/压力验证。

## 6. ESPC52 最终页面树

以下为已实现的对象组织；页面在设备上的实际视觉和交互效果仍未经过硬件验证：

```text
root screen
|- dashboard
|  |- header: C51-style title / readiness / network state
|  |- environment fields: temperature, humidity, pressure, gas, air
|  |- pixel cat: Flash/rodata frame image + state animation
|  `- network status indicator
|- boot readiness overlay
`- voice overlay
```

字段缺失或无效时显示 `--`。Overlay 和 dashboard 的显示/隐藏由 snapshot 和既有 UI state 驱动，不另建业务任务。
