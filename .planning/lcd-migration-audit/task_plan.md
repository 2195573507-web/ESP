# LCD 迁移审计计划

## 目标

以主工作区为目标项目，对比 `分支项目/ESP` 中已可构建的 LCD 实现，审计
WiFi CSI、语音和显示模块的资源竞争，输出迁移对比与分阶段计划；本轮不修改固件。

## 阶段

- [x] 确认工作区、Git 边界和用户既有改动
- [x] 识别 LCD 候选分支与主项目板级目标
- [x] 收集 LCD、CSI、语音的源码和构建配置证据
- [x] 核算 GPIO、总线、任务栈、DMA/PSRAM、内部 RAM、Flash/分区风险
- [x] 写入对比文档和迁移计划文档
- [x] 复核文档中的文件引用、边界与验证门槛

## 约束

- 不修改 ESPC51、ESPC52、ESPS3 或 ESP-server 的实现代码和配置。
- 不启动服务器、不烧录、不将静态分析或构建结果表述为实机验收。
- 保留工作区中所有既有未提交改动。
